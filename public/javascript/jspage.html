REST Api Conventions are also a necessity, as the products become larger and larger, its important to understand where is what.

The expentance here was more inclined to the knowledge of that the GET ALL function lies on top of everything but here is a small summary of the contents:

  • GET all - The function to fetch all elements
  • GET one - the function to fetch a single element using an ID
  • POST - send request creating a new element
  • PATCH - patching parts of a single element (we use this so we dont, flood the internet and server with unecessary bandwidth), using an ID
  • PUT - patch/update the entirety of a single element, basically recreating it from scratch, using an id.
  • DELETE - Deletes a single element using an ID
